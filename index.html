cs_email_queue — first-time setup

One-time setup (~5 min). After this you just visit the page and sign in with Google — no tokens ever again.

Step 1 — Create a Firebase project

Go to console.firebase.google.com and sign in with your Google account. Click "Add project", name it anything (e.g. cs-email-queue), turn off Google Analytics, then click "Create project".

Step 2 — Register a web app & get your config

Inside your new project, click the web icon (</>) near the top. Give it any nickname and click "Register app". Firebase will show you a code block. You need the values from it — paste them into the fields below.

Step 3 — Enable Firestore database

In the left sidebar click Build → Firestore Database → Create database. Choose production mode, pick any region, click Enable. Then click the Rules tab and replace everything with:

rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /{doc=**} {
      allow read, write: if request.auth != null;
    }
  }
}

Click Publish.

Step 4 — Enable Google Sign-In

In the left sidebar click Build → Authentication → Get started. Click Google under Sign-in providers, toggle it On, add your email as the support email, and click Save.

Step 5 — Add your domain to Authorized Domains

Still in Authentication, click the Settings tab → Authorized domains. If you're hosting on GitHub Pages your domain will look like yourusername.github.io — add it here. (localhost is already there for testing.)

Paste your Firebase config values

From the code Firebase showed you in Step 2, copy each value below. They are long strings — just copy everything inside the quotes.
